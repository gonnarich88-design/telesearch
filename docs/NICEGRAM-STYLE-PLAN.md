# ทำ Telesearch ให้เป็นแบบ Nicegram Hub

## สิ่งที่ Nicegram Hub มี (จาก https://nicegram.app/hub)

### 1. โครงสร้างหน้าและเมนู
- **เมนูบน:** Channels (All, Categories, Top-100) | Groups (All, Categories, Top-100) | Stickers
- **หน้าแรก (Hub):**
  - Hero: ป้ายใหญ่ + คำอธิบาย + search
  - **Browse By Category** — การ์ดหมวดหมู่ (Blogs, News, Tech, ฯลฯ) แสดงจำนวนช่อง + สมาชิกรวม
  - **Top 100** — รายการช่อง/กลุ่มเรียงตามจำนวนสมาชิก
  - **Telegram Channels** — รายการช่อง + ปุ่ม "View More"
  - **Telegram Groups** — รายการกลุ่ม + ปุ่ม "View More"
  - **Sticker Packs** — รายการสติกเกอร์ (ถ้าต้องการ)
- **หน้าย่อย:** `/channels`, `/groups`, `/categories`, `/category/[slug]`, `/channel/[username]`, `/group/[username]`, `/top`, `/stickers`

### 2. ข้อมูลที่แสดง
- ชื่อช่อง/กลุ่ม, ลิงก์ t.me, จำนวนสมาชิก (อัปเดตจริง), คำอธิบายสั้น
- หมวดหมู่มีสถิติ: จำนวนช่องในหมวด + สมาชิกรวม

### 3. แหล่งข้อมูลของ Nicegram
- น่าจะดึงจาก Telegram จริง (หรือให้คนส่งมาแล้วดึงสถิติ) จึงมีข้อมูลจำนวนมาก (หลายล้านช่อง)

---

## สิ่งที่ต้องทำใน Telesearch

### Phase 1: หน้าและ UI แบบ Hub (ไม่ต้องมีข้อมูลเยอะก่อน)

| ลำดับ | รายการ | รายละเอียด |
|-------|--------|------------|
| 1 | **Layout + เมนู** | เมนูบน: ช่องทาง | กลุ่ม | บอท, ลิงก์ไป /channels, /groups, /bots, /categories |
| 2 | **หน้าแรก (/)** | Hero + ช่องค้นหา, บล็อก "เลือกตามหมวดหมู่" (การ์ดหมวด), บล็อก "ยอดนิยม" (เรียงตามจำนวนคน), บล็อก "ช่องทางล่าสุด" / "กลุ่มล่าสุด" |
| 3 | **หน้ารายการ** | /channels, /groups, /bots — แสดงรายการแบบตาราง/การ์ด + ตัวกรองหมวด + search |
| 4 | **หน้ารายละเอียด** | /channel/[id], /group/[id], /bot/[id] — แสดงชื่อ, ลิงก์, จำนวนคน, อัปเดตล่าสุด, หมวดหมู่ (ดีต่อ SEO) |
| 5 | **หน้าหมวดหมู่** | /categories แสดงการ์ดทุกหมวด, /category/[slug] แสดงช่อง/กลุ่มในหมวดนั้น |
| 6 | **Top / ยอดนิยม** | /top หรือบล็อกในหน้าแรก — เรียงตาม memberCount |

### Phase 2: ข้อมูลให้เหมือนของจริง (ถ้าอยากได้แบบ Nicegram จริงๆ)

| ลำดับ | รายการ | รายละเอียด |
|-------|--------|------------|
| 1 | **ดึงข้อมูลจาก Telegram** | ใช้ Bot API (`getChat`, `getChatMemberCount`) หรือ MTProto — ต้องมี username/link ของช่องหรือกลุ่มก่อน (จากที่ผู้ใช้ส่ง หรือจากรายการที่รวบรวมไว้) |
| 2 | **ให้คนส่งช่อง/กลุ่ม** | ฟอร์ม "เพิ่มช่องทางของคุณ" → เก็บ username/link → ใช้บอทไปดึงชื่อ + จำนวนสมาชิกมาอัปเดตใน DB |
| 3 | **อัปเดตจำนวนสมาชิกเป็นระยะ** | Cron/Job เรียก Telegram API อัปเดต member_count, last_updated_at |

### Phase 3: เพิ่มเติม (ถ้าต้องการ)

- Stickers — เพิ่มประเภท sticker pack ใน entity หรือตารางแยก
- Top 100 — หน้า /top เรียงตาม memberCount (ข้อมูลมีแล้วใน DB)
- Verification — ให้แอดมินช่อง/กลุ่มยืนยันตัว (ต้องมีระบบล็อกอิน/ยืนยัน)

---

## สรุปสั้นๆ

- **ให้เหมือน “แบบเว็บนี้” ด้าน UI:** ทำ Phase 1 (Layout, หน้าแรกแบบ Hub, หน้ารายการ/รายละเอียด/หมวดหมู่, Top).
- **ให้ข้อมูลเยอะและตัวเลขจริงแบบ Nicegram:** ต้องทำ Phase 2 (เชื่อม Telegram API + วิธีรวบรวมรายการช่อง/กลุ่ม).

ถ้าต้องการให้เริ่มทำเป็นขั้นตอนในโค้ด แนะนำเริ่มจาก Phase 1 ข้อ 1–2 (Layout + หน้าแรกแบบ Hub).
